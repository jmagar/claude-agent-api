Session Forking & Checkpoint Features:

  • Inline Checkpoint Markers: Click üîÄ icon in message flow to access options
  • Fork from Checkpoint: Create new branch starting from that point in conversation
  • Restore to Checkpoint: Roll back session, archiving newer messages as fork
  • Sidebar Tree View: Parent sessions with nested forks (indented branches)
  • Checkpoint Badge: Shows number of forks created from a session
  • Fork Metadata: Name, description, starting point, creation timestamp
  • Timeline View: Visual history of all checkpoints with message counts
  • Safe Restore: Archives discarded messages instead of deleting

Use Cases:

  • Try different implementation approaches without losing work
  • Explore "what if" scenarios mid-conversation
  • Compare multiple solutions to the same problem
  • Roll back when agent went down wrong path
  • Preserve successful intermediate states

Technical Implementation:

  • Checkpoints stored as session state snapshots in PostgreSQL
  • Forked sessions share parent messages (copy-on-write)
  • File changes tracked separately for each fork
  • Restore creates archive fork to prevent data loss
