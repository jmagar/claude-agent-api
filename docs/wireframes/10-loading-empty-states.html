Loading & Empty State Patterns:

  • Skeleton Screens: Animated placeholders for session list, messages, and content areas
  • Spinners: Circular loading indicators for initial load and tool execution
  • Streaming Indicators: Blinking cursor/skeleton for live token streaming
  • Empty States: Friendly messages with clear CTAs for:
    • No sessions (first-time user)
    • No search results
    • No MCP servers configured
    • No custom agents
    • No projects in Code mode
  • Error States: Connection failures with retry/settings actions
  • Onboarding: First-time user flow with API key input

Design Principles:

  • Use skeleton screens for predictable content structure (lists, messages)
  • Use spinners for indeterminate loading (initial connection)
  • Empty states should be friendly, informative, and actionable
  • All empty states include primary CTA + optional secondary action
  • Error states distinguish between user error (API key) and system error (connection)
